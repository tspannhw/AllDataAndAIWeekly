## **Featured: Cortex Agent Evaluations & the Agent GPA Framework**

Snowflake shipped a comprehensive evaluation stack for Cortex Agents this week, anchored by the research-backed **Agent GPA (Goal-Plan-Action) framework**.

### Cortex Agent Evaluations (GA)

**Cortex Agent Evaluations** are now generally available - a complete solution for validating agent behavior and performance before and after production deployment. Define a dataset of test queries with expected responses, run evaluations, and receive structured metrics. Four built-in metrics cover tool selection accuracy, tool execution accuracy, answer correctness, and logical consistency. Custom LLM judge metrics let you encode domain-specific quality standards. Every evaluation run is fully traced and integrated with Cortex Agent Monitoring for thread-level and trace-level debugging.

**Read more:** [Cortex Agent Evaluations: Monitor, Measure and Improve Your AI Agents](https://www.snowflake.com/en/engineering-blog/cortex-agent-evaluations/)

### Agent GPA Framework

The **Agent GPA (Goal-Plan-Action) framework**, developed by Snowflake AI Research and available in the open source TruLens library, evaluates agents across goals, plans, and actions. In benchmark testing on the TRAIL/GAIA dataset, Agent GPA judges caught **95% of human-annotated errors** (1.8x improvement over baseline), achieved **86% error localization** (vs 49% for baseline judges), and matched 570 human-annotated failures across all severity levels.

**Read more:** [What's Your Agent's GPA? A Framework for Evaluating AI Agent Reliability](https://www.snowflake.com/en/engineering-blog/ai-agent-evaluation-gpa-framework/)

### Getting Started with Cortex Agent Evaluations

Hands-on guide to running your first agent evaluation - from dataset preparation through metric analysis.

**Read more:** [Getting Started with Cortex Agent Evaluations](https://www.snowflake.com/en/engineering-blog/getting-started-cortex-agent-evaluations/)

## **Featured: Apache Polaris - The Iceberg REST Catalog and the End of Vendor Lock-In**

**Apache Polaris** graduated to an Apache Top-Level Project on February 18, 2026, cementing its role as the vendor-neutral standard for Iceberg catalog management. Snowflake's engineering blog lays out the case clearly: Polaris implements bidirectional Iceberg REST Catalog access with no proprietary extensions, the same open source project powers Snowflake's Horizon Catalog in production, and contributions from hundreds of people across dozens of companies (Dremio, Google Cloud, Microsoft, Confluent, AWS and more) have driven explosive growth. The community has spoken - the Apache Iceberg REST catalog specification is the definitive language for data exchange.

**Read more:** [Apache Polaris: The End of Data Vendor Lock-In](https://www.snowflake.com/en/engineering-blog/apache-polaris-iceberg-rest-catalog/)

## **Featured: PARSE_JSON 2x Faster with simdjson**

Snowflake's **PARSE_JSON** function now runs **2x faster on average** thanks to integration with **simdjson**, a high-performance JSON parser leveraging SIMD instructions. With nearly 200 million queries a day using JSON parsing, this delivers meaningful performance improvements at scale - an average 5.7% latency improvement for queries using PARSE_JSON. The implementation maintains 100% backward compatibility with all Snowflake-specific JSON behaviors. No query rewrites needed. No behavior changes. Just faster execution.

**Read more:** [Turbocharging parse_json() in Snowflake: 2x Faster Performance with simdjson](https://www.snowflake.com/en/engineering-blog/snowflake-parse-json-performance-simdjson/)

### Snowflake Shutdown SQL — Stop All Running Items

Use these SQL commands to suspend all active Snowflake resources and minimize credit consumption:

```sql
-- ============================================
-- STOP ALL SNOWFLAKE ITEMS
-- Run as ACCOUNTADMIN in Snowsight or SnowSQL
-- ============================================

-- 1. Suspend all warehouses
ALTER WAREHOUSE IF EXISTS GHOST_DETECT_WH SUSPEND;
ALTER WAREHOUSE IF EXISTS INGEST SUSPEND;

-- 2. Suspend Cortex Search services
ALTER CORTEX SEARCH SERVICE IF EXISTS GHOST_DETECTION.APP.SPIRIT_BOX_SEARCH SET TARGET_LAG = 'DOWNSTREAM';

-- 3. Suspend all tasks
ALTER TASK IF EXISTS GHOST_DETECTION.APP.REFRESH_GHOST_STATS SUSPEND;

-- 4. Suspend dynamic tables (already suspended, but verify)
-- ALTER DYNAMIC TABLE <db>.<schema>.<table> SUSPEND;

-- 5. Verify nothing is running
SELECT 'Warehouses' AS type, NAME, STATE 
  FROM SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSES 
  WHERE DELETED IS NULL AND STATE = 'STARTED';

SHOW CORTEX SEARCH SERVICES IN GHOST_DETECTION.APP;

SELECT 'Tasks' AS type, NAME, STATE 
  FROM GHOST_DETECTION.INFORMATION_SCHEMA.TASK_HISTORY 
  WHERE STATE = 'STARTED'
  LIMIT 10;
```

Also available via shell: `./manage.sh snowflake-stop`
